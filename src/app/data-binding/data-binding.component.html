Data binding is the syntax used to communicate between the model and the template. Interpolation is a form of data binding, but in this section we'll cover the other data binding syntaxes.

Property binding should be used when you want to set a template element property to a component model attribute value. This is especially useful when you want a template element property to change programatically (see below), or if you want to pass values to a child component (check out the framed-details component in the source code for more).

Event binding should be used whenever you want to respond to a user's action or some other event. A full list of events can be found at this mozilla source. Custom events can be created and emitted using the EventEmitter class.

Two way binding allows two way communication between the component model and the template. Template element properties can be two way bound, but this binding is most useful when working with forms filled out by the user. Forms are bound using the NgModel directive. If you want the page to respond in real time to the user's input, two way binding is the thing for you!
